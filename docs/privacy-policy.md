# ScreenshotSafe Privacy Policy

Effective date: May 17, 2026

ScreenshotSafe is a self-hosted screenshot capture, editing, and sharing tool. The ScreenshotSafe browser extension is designed to connect only to the ScreenshotSafe server URL that you configure.

This policy describes the ScreenshotSafe browser extension and the self-hosted ScreenshotSafe web application.

## Summary

- ScreenshotSafe does not operate a central cloud service.
- The extension developer does not receive, store, sell, or share your screenshots, browsing history, API tokens, or account data.
- Screenshots are sent only to the ScreenshotSafe server URL that you configure.
- Data stored by a self-hosted ScreenshotSafe server is controlled by the person or organization that runs that server.

## Information the Extension Handles

The ScreenshotSafe extension may handle the following information:

- Server URL: the ScreenshotSafe server address you enter in the extension settings.
- API token: the token you create on your ScreenshotSafe server so the extension can upload screenshots.
- Visible-tab screenshot: the image captured when you intentionally use the extension to take a screenshot.
- Page metadata: the page title and source URL associated with the captured screenshot.
- Edited screenshot content: cropped or redacted screenshot data that you choose to upload.

The extension stores the server URL in your browser's local extension storage. Screenshot drafts are held temporarily inside the extension while you review, crop, or redact them before upload.

## How Information Is Used

ScreenshotSafe uses this information to:

- verify that the configured server URL works with your browser sign-in;
- capture the visible tab when you request a screenshot;
- let you crop or redact the screenshot before upload;
- upload the finalized screenshot and metadata to your configured ScreenshotSafe server;
- return share links created by your configured ScreenshotSafe server.

ScreenshotSafe does not use this information for advertising, analytics, profiling, or cross-site tracking.

## Where Data Is Stored

The extension does not send screenshot data to the extension developer or to a ScreenshotSafe-operated cloud service.

When you upload a screenshot, the screenshot image, title, source URL, annotations, crop settings, visibility setting, expiration setting, and related metadata are stored on the ScreenshotSafe server you configured. If you run your own ScreenshotSafe server, that data remains on infrastructure you control. If you use a server run by someone else, that server operator controls storage, access, retention, backups, and logs for that server.

## Sharing

ScreenshotSafe does not sell personal information.

The extension developer does not share your screenshots or account data with third parties because the extension developer does not receive that data.

Screenshots uploaded to a ScreenshotSafe server may be shared through links generated by that server. Unlisted share links are not the same as private authentication: anyone with an unlisted share URL can view that shared image until it expires, is made private, or is deleted.

## Retention and Deletion

The extension keeps the configured server URL until you change or remove it from the extension settings or uninstall the extension.

Temporary screenshot drafts in the extension are short-lived and are used only to support the pre-upload editing flow.

Uploaded screenshots and metadata are retained by the configured ScreenshotSafe server according to that server's settings and the expiration options selected for each screenshot. You can delete screenshots from the ScreenshotSafe web application. Server operators may also maintain logs or backups according to their own hosting practices.

## Permissions

The extension requests browser permissions needed for its core features:

- `activeTab`: to capture the currently active visible tab after you invoke the extension.
- `contextMenus`: to provide screenshot and settings actions from the browser context menu.
- `storage`: to save the configured server URL locally in the browser.
- `scripting`: to show the sign-in prompt in the active tab when your browser session is unavailable.

## Security

Use HTTPS for any ScreenshotSafe server exposed outside local development. The extension uses your browser sign-in session for authenticated requests to the configured server.

## Children's Privacy

ScreenshotSafe is not directed to children and does not knowingly collect personal information from children through any central service.

## Changes to This Policy

This policy may be updated when ScreenshotSafe changes. The effective date above will be updated when material changes are made.

## Contact

For questions about this policy or the ScreenshotSafe project, contact the maintainer or owner of the repository or distribution channel where you obtained ScreenshotSafe.

For questions about data stored on a specific self-hosted ScreenshotSafe server, contact the person or organization that operates that server.
